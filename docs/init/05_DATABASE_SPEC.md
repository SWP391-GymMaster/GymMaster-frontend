# 05 — DATABASE SPEC

**Status:** Approved | **DBMS:** SQL Server | **ORM:** EF Core 10 (Code First) | **Last updated:** 2026-06-02

> Final backend change notes (`CHANGELOG_vs_old_spec.md`, `DB_DIFF_FOR_DBTEAM.md`) supersede the older EF Core 8 / direct `RoleId` draft. Backend/database implementation is external to this frontend repo, but frontend docs and mocks must follow these contracts.

## 1. Nguyên tắc

- Quan hệ rõ, có PK/FK, map 1-1 với use cases (SG-03).
- Bảng dùng `snake_case`; khóa chính canonical là `Id BIGINT IDENTITY(1,1)`.
- Soft delete (`IsDeleted`, `DeletedAt`) cho entity nghiệp vụ.
- Audit cột chuẩn: `CreatedAt`, `UpdatedAt` (UTC) cho mọi bảng chính.
- Mọi thay đổi qua EF Core Migration; backend **connect-only** tới SQL Server, không auto-create/auto-seed database trong runtime.
- API/backend contracts dùng base path `/api/v1/...`.

## 2. Bảng chính (Core)

| Bảng | Vai trò | Quan hệ chính |
|---|---|---|
| `users` | Tài khoản đăng nhập | 1-1 member/trainer profile; n-n roles qua `user_roles` |
| `roles` | Admin/Staff/PT/Member | n-n với `users` qua `user_roles` |
| `user_roles` | Gán role cho user | FK `users`, `roles`; thay thế `Users.RoleId` |
| `refresh_tokens` | Refresh token JWT | n-1 `users` |
| `password_reset_tokens` | Token reset mật khẩu | n-1 `users`; hash token, expiry, used timestamp |
| `member_profiles` | Hồ sơ hội viên | 1-1 `users`; 1-n Memberships, CheckIns |
| `trainer_profiles` | Hồ sơ PT | 1-1 `users`; 1-n TrainerAssignments |
| `membership_packages` | Gói tập mẫu | 1-n Memberships |
| `memberships` | Gói đã gán cho Member | n-1 Members, Packages; 1-n Payments |
| `payments` | Giao dịch thanh toán | n-1 Memberships |
| `check_ins` | Lượt đến phòng | n-1 Members |
| `trainer_assignments` | Phân công PT-Member | n-1 Trainers, Members |
| `workout_plans` | Giáo án | n-1 Members, Trainers; 1-n WorkoutExercises |
| `workout_exercises` | Bài tập trong giáo án | n-1 WorkoutPlans |
| `trainer_notes` | Ghi chú PT theo ngày | n-1 Members, Trainers |
| `progress_logs` | Tiến độ cơ thể | n-1 Members |
| `audit_logs` | Log hành động | n-1 Users (actor) |

## 3. Bảng dinh dưỡng (Core/Secondary)

| Bảng | Vai trò |
|---|---|
| `food_items` | Danh mục món ăn (calo, macro/100g hoặc /khẩu phần) |
| `meal_logs` | Bữa ăn theo ngày của Member |

## 4. Auth/RBAC schema deltas đã chốt

- `users` **không còn `RoleId`**; quyền đi qua `user_roles` để hỗ trợ model RBAC chuẩn hơn.
- `users.Status` dùng string contract `"active" | "locked"` thay vì numeric enum.
- `LockoutUntil` đổi thành `LockedUntil`.
- Thêm `LoginWindowStartedAt` cho rule brute-force trong 15 phút.
- Loại bỏ `RowVersion` khỏi user schema hiện tại.
- `password_reset_tokens` là một phần auth scope để hỗ trợ forgot/reset password.
- Admin create user cho phép password optional; backend có thể trả `initialPassword` và `passwordAutoGenerated`.

## 5. Quan hệ then chốt (ràng buộc nghiệp vụ)

- Member ↔ Membership: 1 Member có nhiều Membership theo thời gian; **tối đa 1 Active** tại 1 thời điểm.
- Member ↔ PT: qua TrainerAssignment; **tối đa 1 active**. Assign mới khi đang active phải trả lỗi nghiệp vụ cho tới khi assignment cũ kết thúc.
- Membership ↔ Payment: 1 membership có ≥1 payment; status Active sau khi có payment hợp lệ.
- MealLog ↔ MealLogItem: 1-n; xóa MealLog → xóa kèm items (cascade nếu backend chọn hard-delete record meal tạm).
- Mutating actions tạo `audit_logs`.

## 6. Ưu tiên triển khai dữ liệu (theo sprint)

1. **Sprint 1:** `users`, `roles`, `user_roles`, `refresh_tokens`, `password_reset_tokens`, `member_profiles`, `trainer_profiles`.
2. **Sprint 2:** packages, memberships, payments, check-ins.
3. **Sprint 3:** trainer assignments, workout plans/exercises, trainer notes, progress logs.
4. **Sprint 3+:** food items, meal logs, meal log items, calorie targets, audit logs.

## 7. Index gợi ý

- `users(Email)` unique.
- `user_roles(UserId, RoleId)` unique.
- `member_profiles(PhoneNumber)` unique when present.
- `memberships(MemberId, Status, EndDate)` cho check-in & dashboard.
- `trainer_assignments(MemberId, Status)` để enforce active PT.
- `check_ins(MemberId, CheckInAt)` cho báo cáo theo ngày.
- `meal_logs(MemberId, LogDate)` và `meal_log_items(MealLogId)` cho calorie summary.
